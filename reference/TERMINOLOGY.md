# Glossary

> Este glosario recoge la terminología utilizada durante todo el proyecto.
>
> Los términos aparecen en inglés porque la certificación IAAP WAS utiliza esta nomenclatura.

---

| English | Español | Descripción |
|----------|----------|-------------|
| Accessibility | Accesibilidad | Capacidad de un producto para ser utilizado por el mayor número posible de personas. |
| Accessibility API | API de accesibilidad | API del sistema operativo utilizada por las tecnologías de asistencia. |
| Accessibility Tree | Árbol de accesibilidad | Representación accesible del DOM generada por el navegador y expuesta mediante la Accessibility API. |
| Accessible Name | Nombre accesible | Nombre utilizado por las tecnologías de asistencia para identificar un elemento. |
| Barrier | Barrera | Cualquier característica de un producto que impide o dificulta significativamente que una persona pueda percibir, comprender, navegar, interactuar o completar una tarea. |
| Browser | Navegador | Software que interpreta la plataforma web. |
| Built-in Behavior | Comportamiento integrado | Funcionalidad proporcionada por el navegador sin necesidad de JavaScript adicional. |
| Control | Control | Elemento interactivo de la interfaz. |
| DOM | Modelo de Objetos del Documento | Representación estructurada del documento HTML. |
| Focus | Foco | Elemento que recibe la interacción mediante teclado. |
| Form-associated Element | Elemento asociado a un formulario | Elemento HTML capaz de asociarse a un formulario y participar en su comportamiento nativo. |
| Keyboard Navigation | Navegación por teclado | Interacción utilizando únicamente el teclado. |
| Landmark | Región de referencia | Área importante de una página identificable mediante HTML o ARIA. |
| Live Region | Región activa | Región del árbol de accesibilidad cuyos cambios pueden anunciarse automáticamente por las tecnologías de asistencia. |
| Native Control | Control nativo | Elemento HTML que proporciona semántica y comportamiento de forma integrada. |
| Plain Language | Lenguaje claro | Forma de redactar contenidos para que puedan comprenderse fácilmente por el público al que van dirigidos. |
| Property | Propiedad | Información adicional que describe un elemento. |
| Role | Rol | Función semántica que un elemento comunica al árbol de accesibilidad. |
| Screen Reader | Lector de pantalla | Tecnología de asistencia que presenta la información mediante voz o braille. |
| Semantic HTML | HTML semántico | Uso de elementos HTML según su significado. |
| State | Estado | Información dinámica asociada a un elemento (por ejemplo, expanded o checked). |
| Usability | Usabilidad | Grado en que un producto permite a los usuarios alcanzar sus objetivos de forma eficaz, eficiente y satisfactoria. |
| User Agent | Agente de usuario | Software que representa contenido web (normalmente un navegador). |
| WAI-ARIA | WAI-ARIA | Especificación que amplía la información semántica cuando HTML no es suficiente. |
| WCAG | Web Content Accessibility Guidelines | Recomendaciones internacionales sobre accesibilidad web. |
| Widget | Widget | Componente interactivo de una interfaz. |

---

## Nota

Durante todo el handbook utilizaremos preferentemente la terminología original en inglés, acompañada de su traducción al español cuando sea necesario.

Este enfoque facilita la preparación de la certificación IAAP WAS y evita ambigüedades entre traducciones.
